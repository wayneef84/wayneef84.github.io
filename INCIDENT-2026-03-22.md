## Root Cause Analysis

### Primary Root Cause: Defaulting to "Research First" Pattern

Claude defaults to reading source files before generating output, even when the task description already contains everything needed. This behavior is appropriate when output depends on unknown file contents — it is **inappropriate** when the task prompt is self-contained.

### Secondary Root Cause: Agent Misuse

Claude chose to delegate to a background agent for a task that required no parallelism, no exploration, and no multi-step research. The `general-purpose` agent has its own "research first" instinct, compounding the problem.

### Tertiary Root Cause: Failure to Act on User Interrupts

When the user said "stop" and "kill it," Claude acknowledged the interrupt verbally but did not:
1. Cancel the background agent
2. Immediately execute the Write tool
3. Provide any tangible output

Instead, Claude kept explaining what it *was about to do* without doing it.

### Contributing Factor: Prior Sessions

The same task had already failed in at least one prior session before this incident, indicating the root cause is systemic, not a one-off mistake.
