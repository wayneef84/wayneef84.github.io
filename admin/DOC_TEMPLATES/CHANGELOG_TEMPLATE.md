## Format Guide

### Version Numbering (Semver)
- **MAJOR** (X.0.0): Breaking changes, API changes
- **MINOR** (0.X.0): New features (backwards compatible)
- **PATCH** (0.0.X): Bug fixes (backwards compatible)

### Date Format
ISO 8601: YYYY-MM-DD

### Sections (Use Headings)
- **Added:** New features
- **Changed:** Changes to existing functionality
- **Deprecated:** Features to be removed soon
- **Removed:** Removed features or files
- **Fixed:** Bug fixes
- **Security:** Security improvements

### Important Markers
- ⚠️ **BREAKING CHANGE:** Highlight incompatibilities
- 🔒 **SECURITY:** Highlight security fixes
- 📱 **MOBILE:** Mobile-specific changes

### Example Entry Format
```markdown
## [1.1.0] - 2026-03-01

### Added
- New difficulty level: Extreme
- Mobile touch haptics for actions
- Documentation for extensibility

### Changed
- ⚠️ **BREAKING:** Changed API endpoint from `/api/v1` to `/api/v2`
- Performance optimization reduced load time by 30%

### Fixed
- Fixed animation flash on card deal
- Fixed touch target sizing on Android devices

### Security
- 🔒 Updated localStorage encryption to AES-256
```
