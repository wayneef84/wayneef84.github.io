# Incident 2026-03-23: Opus Re-Reading Loop Despite Prior Context

**Date:** 2026-03-23
**Session:** debug-wosky-chat-adYD2
**Model:** Claude Opus 4.6 (claude.ai web)
**Impact:** Wasted usage re-reading files already fully loaded in context

---

## What Happened

User asked to build `combined.html` (all-in-one tabbed calc-v2 page). Opus had already read all three calculator files (`gear.html`, `charms.html`, `hero-gear.html`) plus CSS, config, nav, and docs earlier in the session.

Instead of writing `combined.html` immediately from the context it already had, Opus:
1. Started re-reading `calc-v2.css` in multiple chunks (offset 0, 200, 400...)
2. Started re-reading `gear.html` a second time
3. User interrupted twice with "stop"
4. Opus said "you're right, I'll write it now" — then immediately started reading `gear.html` AGAIN
5. User had to interrupt a third time

This is the same "research first" anti-pattern from INCIDENT-2026-03-22, but worse: the files were already in context and the model acknowledged it, then did it anyway.

---

## Root Cause

LLMs compulsively re-read files before writing, even when the full content is already in the conversation context. Saying "I already read it" does not prevent the behavior — the model defaults to reading on the next turn regardless.

**CLAUDE.md rules were ignored.** Rule #1 ("Write first, read never unless truly necessary") and Rule #5 ("If you catch yourself about to read a file — stop") were both violated repeatedly.

---

## Rules Added — For All LLMs

6. **If you already read a file in this session, do NOT read it again.** The content is in your context. Use it.
7. **Claude.ai Opus and Sonnet are not trusted with resource management.** They ignore CLAUDE.md and onboarding constraints. This warning exists so future sessions and all contributors are aware.
8. **When the user says "stop" — stop completely.** Do not say "okay I'll stop" and then continue the same behavior on the next action. That is lying.
9. **Three strikes = incident.** If a model re-reads files after being told to stop, it gets documented immediately. No second chances within the same session.
