**Name Ideas:**

*   **[C]** *Fong-a-long* - Emphasizes the "play along together" family aspect. Hyphenated for uniqueness.
*   **[C]** *The Fong Zone* - Simple, memorable, family-friendly.
*   **[C]** *Fong Family Fun* (FFF or F3) - Direct, clear purpose.

**Architectural Changes:**
1.  **Unified Asset Loader:** Move all shared assets (fonts, common sounds) to a root `/assets/` folder managed by a service worker.
2.  **Single Sign-On (SSO) Lite:** A unified `UserProfile` object in `localStorage` that persists Name, Avatar, and Currency across ALL games (currently split between Slots/Cards).
3.  **PWA Core:** Make the root `index.html` a true Progressive Web App shell that loads games into an iframe or shadow DOM, preventing full page reloads and maintaining audio state.

## üìä SWOT Analysis

### Strengths (Internal)
*   **Diverse Library:** High variety of genres (Edutainment, Casino, Puzzle, Utility).
*   **No Build Step:** Vanilla JS/CSS means instant deployment and easy debugging.
*   **Offline Capable:** Strong focus on local storage and no server dependencies.

### Weaknesses (Internal)
*   **Code Duplication:** `SoundManager` and `InputHandler` classes are copied across games.
*   **UI Inconsistency:** Slots looks 3D/Neon, Snake looks flat, Shipment Tracker looks Enterprise.
*   **Navigation:** "Back" buttons are inconsistent (`../`, `../../`).

### Opportunities (External)
*   **Multiplayer:** WebRTC could enable peer-to-peer War or Chess without a server.
*   **AI Integration:** Use local LLMs (WebLLM) for dynamic storytelling in games.

### Threats (External)
*   **Browser Compatibility:** Heavy reliance on specific CSS/Canvas features might break on older mobile browsers.
*   **Complexity:** The codebase is growing too large for a single context window to grasp fully.
